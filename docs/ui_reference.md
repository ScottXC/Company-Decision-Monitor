# UI 设计参考

## 风格原则

- 参考现代金融终端的信息密度和模块排布。
- 借鉴简洁卡片、市场感导航和范围搜索体验，但不复制任何品牌素材、配色或商标。
- 使用克制的浅色内容区和深色侧栏，减少装饰性渐变和插画。

## 颜色原则

- 主背景：浅灰白，降低长期阅读疲劳。
- 侧栏：深色，提高模块识别度。
- 主操作：绿色，用于搜索、添加、自选等正向动作。
- 信息状态：蓝色。
- 警示状态：琥珀色。

## 信息密度

- 首页信息密度高于普通官网，但避免拥挤。
- 每个区域只展示当前阶段必要信息。
- 空状态必须说明下一步，而不是放置伪造数据。

## 组件规范

- 卡片：用于模块、指标、空状态和设置项。
- 表格：只展示结构占位，不显示虚构行。
- 搜索：顶部全局搜索 + 搜索页大输入框。
- 导航：左侧主导航，页面标题固定在顶栏。
- Badge：用于 UI Preview Mode、状态和未来风险等级。
