# v0.1.1 UI Structure

本文件记录 Company Decision Monitor v0.1.1 UI Polish 的信息架构和展示规则。项目主版本仍为 `v0.1.1`，当前模式仍为 `Public + Free API Network Mode`。

## 主导航

主导航收敛为 5 个入口：

1. 首页
2. 搜索公司
3. 自选公司
4. 公司详情
5. 设置

热门公司、风险监控、AI 总结等能力不作为主导航入口。未完成的能力只保留清晰说明，不展示伪造数据。

## 页面职责

### 首页

首页只回答三个问题：

- 我可以搜索什么？
- 我当前有哪些自选公司？
- 数据源是否大体可用？

首页不展示完整 provider 错误列表，不展示无可靠来源的热门公司榜单。

### 搜索公司

搜索页围绕一个任务：输入公司名、股票代码、简称、缩写、LEI 或注册号并查看结果。

结果分组：

- 最佳匹配
- 上市公司
- 法人实体
- 相关新闻
- 可能相关

局部错误、未配置 key 和 provider 诊断默认折叠在“数据源诊断”中。

### 自选公司

自选页是本机 watchlist 清单。字段控制为：

- 公司
- 标识
- 类型
- 国家 / 市场
- 最近更新时间
- 数据来源
- 操作

删除自选只删除本机 watchlist 项，不删除缓存或历史搜索结果。

### 公司详情

公司详情采用研究卡片结构：

- Header：公司名称、标识、地区、来源和自选操作。
- 关键摘要：公司类型、市场信息、注册信息、新闻状态。
- Tabs：概览、证券信息、法人注册、新闻动态、数据来源。

缺失字段显示“暂无数据”，不补 0，不生成假行情、假财务或假新闻。

### 设置

设置页拆为 6 个 tab：

1. 数据源总览
2. 免费 API key
3. 公开数据源
4. 搜索设置
5. 缓存与隐私
6. 关于软件

API key 输入放在折叠卡里，默认只显示 masked key 和配置状态。

## 视觉层级

- 首页和搜索页优先显示主操作。
- provider 细节只放入设置页或折叠诊断区。
- 主按钮使用统一绿色强调色。
- 危险动作使用红色弱底。
- badge 统一用于状态、类别和 provider 摘要。
- 卡片使用统一圆角、边框和 padding。

## 布局稳定规则

为避免卡片重叠、横向撑爆和字体发虚，v0.1.1 UI Polish 增加以下约束：

- 主内容区禁止横向滚动，长内容必须换行或进入折叠区。
- 指标卡片网格最多两列，不再使用四列硬排。
- 公司详情字段默认单列展示，避免中文长字段挤压。
- 自选公司使用列表卡片，不再使用 7 列宽表格。
- 页面标题和操作按钮分为上下两行，避免标题区横向拥挤。
- 字体使用 14px 基础字号，字重限制在 600 / 700，避免 Qt 渲染不清晰。
- badge 允许换行，不能强行撑宽页面。

## 错误展示规则

用户界面不展示 traceback、异常类名或原始堆栈。

推荐文案：

- 网络请求超时，请稍后重试。
- 该数据源暂时不可用，已继续尝试其他来源。
- 该免费 API key 可能无效，请检查设置页。
- 该免费层可能达到额度限制，请稍后重试。
- 当前没有找到高置信度匹配。

详细错误只允许放入开发者诊断或日志。

## Provider 状态规则

面向用户的状态：

- 可用
- 未配置
- 暂未接入
- 异常
- 限流
- Key 可能无效
- 无结果

首页只显示大类状态：

- 核心搜索源
- 公司信息源
- 新闻源
- 公开补充源

设置页显示完整 provider 表。

## 不做的事

v0.1.1 UI Polish 不新增 provider、不扩展国家数据源、不实现 AI 总结、不实现风险规则引擎、不实现报告导出，不改变已有 API key、搜索、自选、缓存和打包逻辑。
